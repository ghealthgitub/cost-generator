Public API

These endpoints serve cost data to ginger.healthcare. No authentication required — designed to be called from your website frontend or backend.
GET /api/v1/cost-section/{treatment_id}
Returns the approved HTML cost section for a treatment page. Use this to embed cost breakdowns directly on your treatment pages.
ParameterTypeDescription
treatment_idintegerTreatment ID from your treatments table
// Response (approved section exists) { "available": true, "treatment_id": 42, "treatment_name": "Total Knee Replacement", "html": "<div class='cost-section'>...</div>", "updated_at": "2026-03-17T10:30:00" } // Response (not available) { "available": false }
GET /api/v1/cost-estimate?q={procedure_name}
Returns an AI-generated cost estimate for any procedure name. Useful for search results, chatbots, or counselor tools. Uses Claude to generate estimates in real-time.
ParameterTypeDescription
qstringProcedure name (e.g. "CABG", "PET CT Scan", "Robotic Prostatectomy")
// Response { "procedure": "Robotic Prostatectomy", "html": "<div class='estimate-result'>...</div>" }
GET /api/v1/treatments/covered
Lists all treatments that have approved cost sections. Use this to know which treatment pages can display cost data.
// Response [ { "treatment_id": 42, "treatment_name": "Total Knee Replacement", "specialty_name": "Orthopaedics", "updated_at": "2026-03-17T10:30:00" }, ... ]

Integration Example

Here's how to embed cost sections on your treatment pages:
// On your treatment page (JavaScript) const treatmentId = 42; // From your page context const BASE = 'https://costs.ginger.healthcare'; fetch(`${BASE}/api/v1/cost-section/${treatmentId}`) .then(r => r.json()) .then(data => { if (data.available) { document.getElementById('cost-section') .innerHTML = data.html; } });
Note: The cost estimate endpoint (/api/v1/cost-estimate) calls Claude in real-time, so it has a ~3-5 second response time. For production use, consider caching responses. The cost section endpoint returns pre-generated HTML and is instant.
